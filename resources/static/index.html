RAASTA - Ratings as a Service

Out-of-the-box User-Ratings Backend for Developers

RAASTA is a set of high-performing APIs, allowing your team to quickly add User-Ratings support to your applications.

Use cases

  • Insert User-Ratings, as a result of a user-interaction.
  • Retrieve average User-Ratings, to be displayed to users (externally).
  • Retrieve summarized/digested User-Ratings, for your internal use.

Usage

  • Create a new account

    Use the following request to create a new account/product in RAASTA.

    curl -X PUT "https://raasta.heroku.com/api/v1/products?email=[EMAIL]"

    You will receive an email with you ACCESS_TOKEN. Store this ACCESS_TOKEN safely, as it will allow you to make all the following API calls.

  • Store user-ratings

    In your website or app, users will give ratings to your items or services.

    curl -X PUT "https://raasta.heroku.com/api/v1/ratings?token=[ACCESS_TOKEN]&user=[USER_ID]&key1=[ITEM_ID]"

    A User-Rating for item or service [ITEM_ID] will be stored.

  • Retrieve average user-ratings for an item or service

    Retrieve the average user-rating for a specific item, so you can pass the value to your frontend/UI.

    curl -X GET "https://raasta.heroku.com/api/v1/ratings?token=[ACCESS_TOKEN]&key1=[ITEM_ID]"

    Retrieved statistics of User-Ratings for item or service [ITEM_ID] will be returned.
    {
      "message": "stats",
      "params": {
        "key1": [ITEM_ID]
      }
      "stats": {
        "average": [AVERAGE],
        "count": [TOTAL RATINGS],
        "count1": [RATINGS WITH 1-STAR],
        "count2": [RATINGS WITH 2-STAR],
        "count3": [RATINGS WITH 3-STAR],
        "count4": [RATINGS WITH 4-STAR],
        "count5": [RATINGS WITH 5-STAR],
        "stddev": [STANDARD DEVIATION],
        "variance": [VARIANCE]
      }
    }

About RAASTA

Tech Stack and Development Practices

  • Server: NodeJS (v5.7) and Express (v4.x)
    • Mocha (2.x) and Chai (2.x) for TDD
  • Database: Postgres (v9.x) with an Append-Only data model
  • Hosting: Heroku
  • Monitoring: NewRelic APM
  • Logging: Papertrail
  • CI: Wercker

Performance

  • PUT /app/v1/ratings: ~8.4ms
  • GET /app/v1/ratings: ~7.7ms

Motivation

  • Along our careers, we've witnessed multiple teams struggling to solve the user-ratings problem in an effective, efficient way. RAASTA intends to summarize these learnings, by focusing on performance and simplicity.
  • RAASTA was conceived as an experimental project, to measure the Time-To-Market of a new service based on NodeJS+Express, and to test the performance of NodeJS with a Postgres DB.
