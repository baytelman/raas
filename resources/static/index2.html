RAASTA - Ratings as a Service

Out-of-the-box Ratings Backend for Developers

RAASTA is a set of high-performance APIs, allowing your team to quickly add User-based Ratings support to your applications.

RAASTA (v1.0) has been designed for these 3 main use-cases:

  • User-facing: Track specific user's feedback as rating about your application or a specific item/service you provided. Example: "User '123' is providing feedback for DeliveryOrder '345', specifically about the Super Burrito (item id '456') from Taqueria La Mejor (restaurant id '567').
  • User-facing: Retrieve item/service's ratings, to display in your UX. Example: "Using your iOS app, user '987' opens the Detail view of the Super Burrito (item id '456') from Taqueria La Mejor (restaurant id '567'). The user is presented with the average rating for Taqueria La Mejor (4.3 stars) and with the specific rating for the Super Burrito (4.6 stars).
  • Internal: Retrieve summarized/diggested ratings, for your entire account or given a specific item/service. Example: "The Marketing manager of your startup is reviewing the detailed rating history of Taqueria La Mejor (restaurant id '567').

Usage

  • Create a new account

    Use the following request to create a new account/product in RAASTA.

    PUT https://raasta.heroku.com/api/v1/products?email=[EMAIL]

    You will receive an email with you ACCESS_TOKEN. Store this ACCESS_TOKEN safely, as it will allow you to make all the following API calls.

  • Store ratings

    In your website or apps, your users will give ratings to your items or services.

    PUT https://raasta.heroku.com/api/v1/ratings?token=[EMAIL]&user=[USER_ID]

    You will receive an email with you ACCESS_TOKEN. Store this ACCESS_TOKEN safely, as it will allow you to make all the following API calls.
